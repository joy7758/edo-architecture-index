# Architecture

## Layer Definitions

- Layer 0 (Kernel): executable minimal object model with deterministic transitions, deterministic drift gate, and replay-verifiable checkpoint chain.
- Layer 1 (Specification): formal model definitions and semantic rules derived from Layer 0 invariants.
- Layer 2 (Applications): implementation repositories that apply Layer 1 semantics to concrete systems.

## Cross-Layer Invariants

- Determinism: Layer 0 execution is deterministic for fixed input.
- Causal Closure: accepted states must be generated by declared transitions and gate constraints.
- Replay Verifiability: independent recomputation must reproduce checkpoint outputs.
- Non-Expansion of Layer 0: no functional growth inside MVK beyond structural verification scope.
- One-Way Influence: Layer 0 constrains Layer 1/2; Layer 2 requirements cannot redefine Layer 0 invariants.

## Interface Contract

- Layer 0 exposes verifiable artifacts and deterministic behavior.
- Layer 1 references Layer 0 as normative kernel.
- Layer 2 references Layer 1 and may add domain logic without altering Layer 0 rules.

## Acceptance Criteria for New Work

- If a change modifies kernel invariants, it is rejected from Layer 0.
- If a change adds semantics, it belongs in Layer 1.
- If a change adds domain behavior, it belongs in Layer 2.
