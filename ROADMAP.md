# Roadmap

## Phase I: Vienna Preparation (Pre-Event)

- Freeze `fdo-kernel-mvk` at a citable tag (`v0.1.0`).
- Align `pFDO-Specification` terminology with Layer 0 invariants.
- Ensure application repositories include explicit references to Layer 1 specifications.
- Standardize entry-point links from each repository README to this index.

## Phase II: Vienna Delivery (Event Window)

- Demonstrate Layer 0 replay verification live (`demo.py` + `verify.py`).
- Present Layer 1 as formal extension boundary, not executable kernel mutation.
- Show Layer 2 mapping as independent application tracks.

## Phase III: Post-Vienna Consolidation

- Publish a stable architecture snapshot and changelog by layer.
- Add cross-repo traceability matrix from Layer 2 features to Layer 1 clauses.
- Keep Layer 0 unchanged unless verification correctness defects are found.
